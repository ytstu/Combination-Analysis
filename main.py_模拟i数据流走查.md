# `main.py` 默认输入数据流走查

这份文档的目标只有一件事:

用一个真实可用的模拟输入文件，带你按顺序看清楚 `main.py` 是怎么把 Excel 读进来、筛选、拆分、查库、拼接，再变成最终导出结果的。

我已经在项目里放了一份程序会直接读取的默认输入文件:

- `data/input/模拟-i输入-讲解.xlsx`

你之后如果想自己试，可以用这条命令:

```bash
python3 main.py
```

但要注意:

按当前代码，程序会把结果写到 `data/output/组合装数据MMDD.xlsx`。如果在 **2026-03-04** 这一天运行，目标文件就是 `data/output/组合装数据0304.xlsx`，而这个文件现在项目里已经存在，所以直接运行会覆盖它。

为了避免覆盖现有文件，下面这份讲解是用和 `main.py` 相同的处理逻辑做的验证，没有直接调用默认导出。

---

## 1. 先看这个默认输入文件到底是什么

`main.py` 只要求输入 Excel 里有一列:

- `原始商品编码`

我准备的示例输入就是这一列，内容如下:

| 行号 | 原始商品编码 | 这行的作用 |
| --- | --- | --- |
| 1 | `FZ68-0*2` | 正常案例: 基础商品存在，应该能生成组合 |
| 2 | `FZ13-14mm*3` | 正常案例: 倍数更明显，方便看数量计算 |
| 3 | `@HDS38-10mm*2` | 已经在组合资料库里，应该被过滤掉 |
| 4 | `MISSING*4` | 基础商品不存在，但代码仍会保留并导出 |
| 5 | `FZ68-0` | 不符合“组合编码格式”，应该被过滤掉 |
| 6 | `FZ68-0*2` | 和第 1 行重复，用来演示去重 |

你可以把它理解成:

“我给程序一张待检查的商品编码清单，请程序看看哪些是新的组合装候选项。”

---

## 2. 程序入口从哪里开始

入口是:

- `main()`
- `run()`

`run()` 做了 4 件事:

1. 读取固定默认输入文件路径 `data/input/模拟-i输入-讲解.xlsx`
2. 创建 `ExcelDataService`
3. 读取资料库和输入文件，并调用核心处理逻辑
4. 把结果导出成新的 Excel

如果你只想抓主线，记住这一句就够了:

> `run()` 负责串流程，真正的数据加工基本都在 `ExcelDataService.process_data()`

---

## 3. 数据是怎么流动的

下面按代码真正执行的顺序走一遍。

### 第 1 步: 读取输入 Excel

代码:

- `service.load_input_file(input_path)`

它现在只做一件事:

1. `pd.read_excel(file_path)` 读入 Excel

代码不再提前检查有没有 `原始商品编码` 这一列。

如果没有这列，程序会在下一步的去重阶段自然报错，因为那里会直接使用:

```python
df = input_df.copy().drop_duplicates(subset=["原始商品编码"])
```

所以，这一列依然是整个程序的硬性入口，只是现在不单独做前置校验。

---

### 第 2 步: 先按 `原始商品编码` 去重

代码:

```python
df = input_df.copy().drop_duplicates(subset=["原始商品编码"])
```

我们的原始输入有 6 行，但其中 `FZ68-0*2` 出现了两次。

去重后剩下 5 行:

| 原始商品编码 |
| --- |
| `FZ68-0*2` |
| `FZ13-14mm*3` |
| `@HDS38-10mm*2` |
| `MISSING*4` |
| `FZ68-0` |

这里的意思很简单:

同一个编码，不管出现多少次，程序只处理一次。

---

### 第 3 步: 只保留符合“`*数字` 结尾”的编码

这是一个很关键的细节。

代码不是简单判断“有没有 `*`”，而是用了这个正则:

```python
r"^(.*)\*(\d+)$"
```

它的真实含义是:

- 前面可以是任意字符
- 末尾必须是 `*`
- `*` 后面必须是纯数字

也就是说，只有像下面这样的才算有效:

- `FZ68-0*2`
- `FZ13-14mm*3`
- `MISSING*4`

而 `FZ68-0` 会被过滤，因为它没有 `*数字` 结尾。

所以这一轮之后，保留 4 行:

| 原始商品编码 | 是否匹配 `*数字` |
| --- | --- |
| `FZ68-0*2` | 是 |
| `FZ13-14mm*3` | 是 |
| `@HDS38-10mm*2` | 是 |
| `MISSING*4` | 是 |

被过滤掉:

- `FZ68-0`

---

### 第 4 步: 先判断“这条编码是不是已经存在”

这里程序会加载两张资料库:

- 商品资料库: `data/input/商品资料.xlsx`
- 组合资料库: `data/input/组合资料.xlsx`

然后做两个判断:

1. `原始商品编码` 是否已经出现在商品资料库的 `商品编码`
2. `原始商品编码` 是否已经出现在组合资料库的 `组合商品编码`

注意，这里检查的是 **完整的原始编码**，不是拆开后的基础商品编码。

这点非常重要。

例如:

- `FZ68-0*2` 的基础商品 `FZ68-0` 在商品资料库里是存在的
- 但程序这里检查的是 `FZ68-0*2` 这整串
- 所以它的 `单品是否存在` 仍然是空

我们的 4 行在这一轮的判断结果是:

| 原始商品编码 | 单品是否存在 | 组合是否存在 | 说明 |
| --- | --- | --- | --- |
| `FZ68-0*2` | 空 | 空 | 完整编码不在商品库，也不在组合库 |
| `FZ13-14mm*3` | 空 | 空 | 完整编码不在商品库，也不在组合库 |
| `@HDS38-10mm*2` | 空 | `存在` | 这条完整编码已经在组合库中 |
| `MISSING*4` | 空 | 空 | 两边都找不到 |

---

### 第 5 步: 只保留“商品库和组合库里都不存在”的记录

代码:

```python
missing_mask = ~product_exists & ~combo_exists
df = df[missing_mask].copy()
```

这一步的含义是:

只留下“新候选项”。

所以:

- `@HDS38-10mm*2` 因为已经在组合资料库中，被过滤掉

这一轮之后，还剩 3 行:

| 原始商品编码 |
| --- |
| `FZ68-0*2` |
| `FZ13-14mm*3` |
| `MISSING*4` |

这也是为什么你在 `processed_df` 里看到的 `单品是否存在` 和 `组合是否存在` 基本都会是空白:

因为只要其中任意一个是“存在”，那条记录早就已经被过滤掉了。

---

### 第 6 步: 把编码拆成“基础商品编码 + 倍数”

代码里拆完后，会写入两列:

- `倍数前`
- `倍数后`

拆分结果如下:

| 原始商品编码 | 倍数前 | 倍数后 |
| --- | --- | --- |
| `FZ68-0*2` | `FZ68-0` | `2` |
| `FZ13-14mm*3` | `FZ13-14mm` | `3` |
| `MISSING*4` | `MISSING` | `4` |

这一步之后，程序才真正拿到了它后面要查商品资料库的“基础商品编码”。

---

### 第 7 步: 用 `倍数前` 去商品资料库查基础商品

这一段的核心动作是:

- 把商品资料库按 `商品编码` 建成索引
- 用 `倍数前` 去查对应商品

程序尝试读取这些字段:

- `尺寸规格(mm)`
- `颜色`
- `数量(pcs)`

我们的两条能查到商品的记录，查到的真实基础信息如下:

| 倍数前 | 尺寸规格(mm) | 颜色 | 数量(pcs) |
| --- | --- | --- | --- |
| `FZ68-0` | `20` | `混色` | `1` |
| `FZ13-14mm` | `14` | `kc金` | `10` |

而 `MISSING` 查不到，所以这些字段都是空。

---

### 第 8 步: 计算组合后的总件数

代码逻辑:

```python
总件数 = 数量(pcs) * 倍数后
```

按我们的样例:

| 倍数前 | 数量(pcs) | 倍数后 | 计算结果 |
| --- | --- | --- | --- |
| `FZ68-0` | `1` | `2` | `2` |
| `FZ13-14mm` | `10` | `3` | `30` |
| `MISSING` | 查不到 | `4` | 空 |

所以这一步得到的 `pcs_values` 是:

- `FZ68-0*2` -> `2`
- `FZ13-14mm*3` -> `30`
- `MISSING*4` -> 空

---

### 第 9 步: 拼接 `组合商品名称`

程序的拼接公式是:

```text
尺寸规格(mm) + (数量(pcs) * 倍数后) + 颜色
```

于是三条样例变成:

| 原始商品编码 | 实际拼接过程 | 组合商品名称 |
| --- | --- | --- |
| `FZ68-0*2` | `"20" + "2" + "混色"` | `202混色` |
| `FZ13-14mm*3` | `"14" + "30" + "kc金"` | `1430kc金` |
| `MISSING*4` | `"" + "" + ""` | 空 |

这也解释了为什么现有输出里会出现看起来像 `202混色` 这种名字:

不是程序算错了，而是当前逻辑本来就只拼接规格、数量和颜色。

---

### 第 10 步: 组装最终导出表

最后一步在:

- `build_export_df(processed_df)`

它不会保留所有中间列，只会输出这 4 列:

- `组合商品编码`
- `组合商品名称`
- `商品编码`
- `数量`

对应关系是:

| 导出列 | 来源 |
| --- | --- |
| `组合商品编码` | `原始商品编码` |
| `组合商品名称` | `组合商品名称` |
| `商品编码` | `倍数前` |
| `数量` | `倍数后` |

所以最终导出结果就是:

| 组合商品编码 | 组合商品名称 | 商品编码 | 数量 |
| --- | --- | --- | --- |
| `FZ68-0*2` | `202混色` | `FZ68-0` | `2` |
| `FZ13-14mm*3` | `1430kc金` | `FZ13-14mm` | `3` |
| `MISSING*4` | 空 | `MISSING` | `4` |

---

## 4. 把整条链路压缩成一句人话

这个程序本质上是在做:

1. 从输入表里拿到一批“看起来像组合装”的编码
2. 只保留新的候选项
3. 把编码拆成“基础商品 + 倍数”
4. 用基础商品去商品资料库补信息
5. 生成组合商品名称
6. 导出成组合资料格式

如果用我们这份示例来理解，就是:

- `FZ68-0*2` 被识别成“`FZ68-0` 这个基础商品，做 2 倍组合”
- 程序查到 `FZ68-0` 的规格是 `20`，颜色是 `混色`，单个数量是 `1`
- 所以算出总件数是 `2`
- 最后拼出 `202混色`

## 5. 读代码时最容易误解的 5 个点

### 误解 1: 只要有 `*` 就会被处理

不对。

真实规则是:

- 必须匹配 `^(.*)\\*(\\d+)$`
- 也就是必须以 `*数字` 结尾

---

### 误解 2: `单品是否存在` 检查的是基础商品

不对。

它检查的是完整的 `原始商品编码`。

所以 `FZ68-0*2` 的基础商品 `FZ68-0` 明明存在，但 `单品是否存在` 仍然可能是空。

---

### 误解 3: 基础商品查不到就不会导出

不对。

`MISSING*4` 这个例子说明:

- 即使 `MISSING` 在商品资料库里不存在
- 只要这条完整编码不在商品库和组合库里
- 它仍然会被保留到最终导出

只是它的 `组合商品名称` 很可能是空。

---

### 误解 4: `组合商品名称` 还会取商品名称

不对。

当前逻辑已经不再读取 `商品名称` 列。

这就是为什么输出结果就是“规格 + 数量 + 颜色”。

---

### 误解 5: `processed_df` 里那些中间列都会导出

不对。

像这些列:

- `单品是否存在`
- `组合是否存在`
- `倍数前`
- `倍数后`

都只是处理中间状态。

真正写出到 Excel 的只有 4 列:

- `组合商品编码`
- `组合商品名称`
- `商品编码`
- `数量`

---

## 6. 对照函数名，快速定位代码

如果你现在回去看 `main.py`，可以按这个顺序读:

1. `run()`
   看默认输入文件路径和总流程怎么串起来
2. `ExcelDataService.load_databases()`
   看默认读哪两张资料库
3. `ExcelDataService.load_input_file()`
   看输入表是怎么被读进来的
4. `ExcelDataService.process_data()`
   看所有核心业务逻辑
5. `ExcelDataService.build_export_df()`
   看最后导出字段怎么映射

如果你只打算精读一个函数，那就读:

- `ExcelDataService.process_data()`

因为几乎所有“为什么结果是这样”的答案，都在这里。

## 8. 一句话结论

`main.py` 并不是在“管理完整商品信息”，它更像是在做一件很具体的事:

把输入表里的“新组合装候选编码”拆开，去现有商品资料里补能补的信息，然后整理成一张可导出的组合资料表。
